üîå Step 1: Connect to Your EC2

For Amazon Linux / Ubuntu:

# Amazon Linux ssh -i your-key.pem ec2-user@your-ec2-ip # Ubuntu ssh -i your-key.pem ubuntu@your-ec2-ip

üí° Tip: Replace your-key.pem with your actual key file name and your-ec2-ip with your instance's public IP or DNS.

‚ö†Ô∏è Permission Error? Run: chmod 400 your-key.pem

üë§ Step 2: Create a New User

Basic user creation:

# Create a new user named 'john' sudo adduser john # You'll be prompted to set a password and user details # Press Enter to skip optional fields

What happens:

  • Creates a new user account
  • Creates a home directory: /home/john
  • Sets up basic user configuration
  • Prompts for password (if using password auth)

üìù Alternative: Use sudo useradd john for basic user creation without prompts, then set password with sudo passwd john

‚ö° Step 3: Grant Sudo Access (Optional)

If the new user needs administrative privileges:

Method 1: Add to sudo group

# Amazon Linux / Ubuntu sudo usermod -aG sudo john # CentOS / RHEL sudo usermod -aG wheel john

Method 2: Edit sudoers file

# Open sudoers file safely sudo visudo # Add this line at the end: john ALL=(ALL) NOPASSWD:ALL

‚ö†Ô∏è Warning: NOPASSWD allows sudo without password. Remove it if you want password prompts for sudo commands.

üîë Step 4: Setup SSH Key Authentication

Option A: Generate key on local machine

# On your local computer ssh-keygen -t rsa -b 4096 -C "john@company.com" # Save as: john_ec2_key # This creates john_ec2_key (private) and john_ec2_key.pub (public)

Option B: Copy existing public key

# View your public key cat ~/.ssh/id_rsa.pub # Copy the output

üí° Best Practice: Always use SSH key authentication instead of passwords for better security!

üîê Step 5: Add SSH Key to New User

# Switch to the new user sudo su - john # Create .ssh directory mkdir -p ~/.ssh chmod 700 ~/.ssh # Create authorized_keys file touch ~/.ssh/authorized_keys chmod 600 ~/.ssh/authorized_keys # Add public key (paste the public key content) nano ~/.ssh/authorized_keys # Paste the public key, save and exit (Ctrl+X, Y, Enter) # Exit back to your original user exit

‚ö†Ô∏è Critical: Permissions must be exact!
~/.ssh must be 700
~/.ssh/authorized_keys must be 600

‚úÖ Step 6: Test the New User

From your local machine:

# Test SSH connection with the new user ssh -i john_ec2_key john@your-ec2-ip # Or if using default key ssh john@your-ec2-ip

If successful, you should see:

john@ip-xxx-xx-xx-xx:~$

Test sudo access (if granted):

# Try a sudo command sudo apt update # Should work without errors

üéâ Success! Your new user is ready to use!

üîí Security Best Practices

1. Disable Password Authentication

sudo nano /etc/ssh/sshd_config # Set these values: PasswordAuthentication no PubkeyAuthentication yes # Restart SSH service sudo systemctl restart sshd

2. Disable Root Login

# In /etc/ssh/sshd_config PermitRootLogin no

3. Other Tips

  • Use strong SSH keys (4096-bit RSA or Ed25519)
  • Regularly update your EC2 instance
  • Use Security Groups to restrict SSH access to specific IPs
  • Enable AWS CloudTrail for auditing
  • Consider using AWS Systems Manager Session Manager
